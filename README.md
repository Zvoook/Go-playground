# Go Playground

Лаборатория для изучения Go (Golang), отработки навыков конкурентного программирования и разбора паттернов проектирования.

Каждая директория содержит изолированные примеры, демонстрирующие конкретные механизмы языка и подходы к решению задач.

---

## 🛠️ Структура проекта и реализуемые паттерны

### 🧵 Concurrency
Файлы в папке `Concurrency/` демонстрируют работу с горутинами, каналами и примитивами синхронизации. Описания ниже соответствуют **реальной логике** исходного кода:

* **`concurrent_counter_of_N_numbers.go`**
  Параллельное суммирование элементов массива (подход Map-Reduce). Массив делится на чанки, каждый обрабатывается отдельной горутиной, а результаты собираются через канал.
  *Примечание: атомарные операции (`sync/atomic`) в данном коде не используются.*

* **`concurrent_console_logger.go`**
  Сериализация вывода в консоль с использованием канала как мьютекса (Token Passing). Демонстрирует передачу права на запись через канал емкостью 1, а не паттерн Worker Pool или Fan-In.

* **`concurrent_counter_of_lines_in_file.go`**
  Паттерн Fan-In. Параллельное открытие и чтение множества файлов, подсчет строк в каждом и агрегация результатов в одной управляющей горутине через канал.

* **`concurrent_counter_frequency_of_words_in_file.go`**
  Chunk processing для подсчета частоты символов.
  *Примечание: несмотря на название, файл работает с оперативной памятью (генерирует срез `[]rune` и делит его на части), чтение реальных файлов не производится.*

* **`concurrent_searcher_of_pattern_matches_in_directory.go`**
  Параллельный поиск вхождения подстроки в наборе файлов с использованием Fan-In для сбора результатов.
  *Примечание: текущая реализация запускает горутину на каждый файл (unbounded concurrency). Ограничение конкурентности (semaphore) здесь не реализовано.*

* **`concurrent_race_game.go`**
  Моделирование "соревновательной" гонки (Race) с использованием `context.WithTimeout`. Несколько воркеров выполняют задачу с разной задержкой, и первый успешно завершившийся объявляется победителем.
  *Примечание: этот файл не демонстрирует устранение data race (состояния гонки данных) и не использует `sync.Mutex`.*

* **`handwrite_singleflight_pattern.go`**
  Корректная кастомная реализация паттерна `singleflight`. Группирует идентичные запросы, выполняя тяжелую операцию только один раз и возвращая результат ожидающим горутинам через `sync.WaitGroup` и `sync.Mutex`.

* **`utilits.go`**
  Вспомогательные функции для подготовки тестового окружения: создание директории и генерация файлов со случайным содержимым.

* **`main.go`**
  Точка входа. В текущей версии инициирует демонстрацию поиска паттерна (`Search`).

## 🚀 Как запустить

1. Склонируйте репозиторий:
   ```bash
   git clone https://github.com/Zvoook/Go-playground.git
   cd Go-playground/Concurrency

2. Запустите основной сценарий (поиск паттерна в сгенерированных тестовых файлах):
```bash
    go run .
```

Проверка на Data Race (актуально для файлов с разделяемым состоянием):
```bash
    go run -race .
```

🎯 Планы по развитию репозитория
Concurrency Control: Добавление примера с ограниченной конкурентностью (Bounded Concurrency) через семафоры на каналах или golang.org/x/sync/errgroup с SetLimit.

Generics: Рефакторинг существующих решений (например, singleflight или структур Pair) с использованием type parameters для устранения дублирования кода.
Profiling: Добавление бенчмарков (go test -bench) и анализ использования CPU/памяти через pprof и go tool trace.
